title: "Data handling"

---

How you handle data has legal, contractual, and reputational consequences for Mintlify. These guidelines apply to all employees and contractors.

## Data classification

| Classification | Examples | Handling |
|---------------|----------|----------|
| **Public** | mintlify.com content, public docs | No restrictions |
| **Internal** | Meeting notes, roadmaps, Linear issues | Don't share externally without approval |
| **Confidential** | Customer data, contracts, financial data | Encrypt in transit and at rest, need-to-know only |
| **Restricted** | Credentials, API keys, M&A information | Strict access controls, report any exposure immediately |

When in doubt, treat data as one level higher than you think it is.

## Storage rules

- **Approved tools only** — Use Google Drive, Notion, or GitHub for company data. Don't store Mintlify data in personal Dropbox, iCloud, or similar.
- **No local copies of customer data** — Customer data must stay in approved systems. Don't download it to your laptop for analysis — use authorized query tools instead.
- **Credentials are never stored in code** — Use 1Password or environment variables managed through Vercel. If you find credentials in a codebase, rotate them immediately and flag in `#general`.

## Sharing data

- **Internal** — Use the appropriate tool for the audience. Don't forward internal threads to personal email.
- **External** — Confidential data shared with vendors must be covered by an NDA. Check with founders if unsure.
- **Customer data** — Never share customer data with third parties without a data processing agreement in place. Route any customer data requests through the founders and legal counsel.

## Retention and deletion

Don't retain data longer than necessary. When a customer offboards, follow our data deletion process — contact the founders for guidance.

## Reporting a data incident

If you accidentally expose, share, or lose access to confidential or restricted data, report it to [security@mintlify.com](mailto:security@mintlify.com) and post in `#general` immediately. Include what happened, what data was involved, and who may have seen it. Early reporting reduces harm — there is no penalty for honest mistakes reported promptly.
