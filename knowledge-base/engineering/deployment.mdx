title: "Deployment"
description: "How code goes from a merged PR to production at Mintlify, and how to roll back when something goes wrong."
---

Mintlify uses a trunk-based deployment model. Merging to `main` triggers a preview deploy automatically. Production deploys go through Vercel and GitHub Actions.

## Flow overview

```
feature branch → PR → review → merge to main → auto-deploy to preview → promote to production
```

## Deploying to preview

Every merge to `main` triggers a Vercel preview deployment automatically via GitHub Actions. Watch the `#deploys` Slack channel for status. Preview deploys take approximately 2–5 minutes.

If a deploy fails, CI posts the error in `#deploys` with a link to the failed GitHub Actions run.

## Pull request previews

Every PR gets its own preview URL automatically — Mintlify builds it and posts the link as a status check on the PR. This is the primary way to review docs and UI changes before merging.

## Promoting to production

Once a change is verified in preview:

1. Go to the Vercel dashboard (link in `#deploys` channel description).
2. Select the build you want to promote.
3. Click **Promote to Production**.
4. Monitor `#deploys` for confirmation.

Production deploys require at least one other engineer to have reviewed the PR. Use judgment in an emergency.

## Feature flags

New features that need a controlled rollout are deployed behind a feature flag. Ask in `#eng` about the current flagging approach and naming convention.

## Rollbacks

If a production deploy causes an incident:

1. Use the Vercel dashboard to roll back to the previous production deployment immediately.
2. Don't wait to diagnose — roll back first, investigate after.
3. File an [incident](/engineering/incident-response) and start a postmortem if customers were affected.

Rolling back takes about 1–2 minutes. It's safe and reversible — do it early.

## Deploy freeze windows

We freeze production deploys during major launches or holiday periods. The `#deploys` channel will pin a message when a freeze is active.
