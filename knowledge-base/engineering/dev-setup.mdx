This guide assumes a fresh MacBook with MDM enrolled and your `@mintlify.com` accounts active. Complete [laptop setup](/onboarding/tools/laptop-setup) and [account access](/onboarding/tools/account-access) first.

## Prerequisites

- GitHub access — accept the Mintlify org invite in your email
- [Tailscale](/onboarding/tools/vpn-setup) connected
- Node.js 18+ and npm — install via [nvm](https://github.com/nvm-sh/nvm) or [Homebrew](https://brew.sh)

## Step 1: Clone the repository

```bash
mkdir ~/code && cd ~/code
git clone git@github.com:mintlify/mintlify.git
cd mintlify
```

If `git clone` fails with a permission error, your SSH key isn't authorized on GitHub. Add your SSH public key at [github.com/settings/keys](https://github.com/settings/keys), then try again.

## Step 2: Set up environment variables

Each package has a `.env.example` in its root. Copy it:

```bash
cp .env.example .env.local
```

Fill in the required values — ask in `#eng` on Slack for development secrets, or check the 1Password vault named "Engineering / Dev Secrets". Do not commit `.env.local`.

## Step 3: Install dependencies

```bash
npm install
```

The repo uses npm workspaces. This installs dependencies for all packages.

## Step 4: Run the development server

```bash
npm run dev
```

The app should be running at `http://localhost:3000`. Check the terminal output for other service ports (API, worker, etc.).

## Step 5: Preview docs with the Mintlify CLI

To preview docs changes locally using the Mintlify CLI:

```bash
npm i -g mintlify
cd path/to/docs
mintlify dev
```

Docs preview runs at `http://localhost:3000` by default.

## Step 6: Verify your setup

Run the test suite:

```bash
npm test
```

All tests should pass on a fresh clone. If they don't, post in `#eng` with the error output — don't spend more than 15 minutes debugging solo.

## Getting help

Post in `#eng` on Slack with your OS version, the command you ran, and the full error output. Your tech lead can also do a screen share walkthrough if needed.
