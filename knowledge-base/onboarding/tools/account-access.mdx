Most accounts are provisioned automatically when HR creates your employee record. This page lists what you should have and how to request anything that's missing.

## Auto-provisioned accounts

These accounts are created before your start date. Activate them by logging in with your `@mintlify.com` Google account:

| Tool | Used for |
|------|----------|
| Google Workspace | Email, calendar, Docs, Drive |
| Slack | Team messaging and async collaboration |
| Linear | Issue tracking and project management |
| Notion | Internal docs, meeting notes, and the company wiki |
| GitHub | Source code — accept the org invite in your email |
| 1Password | Password manager — install on your laptop and browser |

## Role-specific accounts

Some tools require a separate request, either because they're role-specific or have limited licenses:

| Tool | Used for | How to request |
|------|----------|---------------|
| Figma | Design and prototyping | Ask the design team in `#design` |
| Vercel | Deployment and preview environments | File an IT access request |
| AWS | Cloud infrastructure | File an IT access request, manager approval required |
| PostHog | Product analytics | File an IT access request |
| Stripe dashboard | Billing and subscription data | Manager + finance approval required |
| Linear Admin | Team and project admin settings | Ask your manager |

## Mintlify product access

You'll also get access to Mintlify itself as a team member:

- A Mintlify account at [mintlify.com](https://mintlify.com) linked to your work email
- Access to internal Mintlify projects used for company docs and the team wiki
- Mintlify CLI: `npm i -g mintlify` — run `mintlify dev` to preview docs locally

## Something missing?

Post in `#general` on Slack with:
- The tool you need
- Why you need it

Your manager will confirm the request. Routine access is fulfilled within one business day.
